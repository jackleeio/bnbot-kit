# 评论曝光预测算法 (Exposure Prediction Algorithm)

> **版本**: v1.2
> **更新日期**: 2026-01-30

## 概述

该算法用于预测用户在某条推文下发表评论后，该评论能够获得的曝光量范围。核心思路是：**评论的曝光 = 推文的剩余流量 × 你能分到的比例 × 账号系数 × 内容加成**。

## 核心公式

```
E_comment = (V_future × R_rank) × B_gate × A_health × M_media + (V_now × K_scroll)

其中：
- V_future: 推文未来增量（还会新增多少阅读）
- R_rank: 排名转化率（你的评论能分到多少）
- B_gate: 蓝标门槛系数（蓝标=1.0，非蓝标=0.15）  [v1.1]
- A_health: 账号健康度（根据账号年龄 0.1~1.0）  [v1.1]
- M_media: 媒体类型加成（视频=1.5，图片=1.2，无=1.0）  [v1.2 新增]
- V_now: 当前存量曝光
- K_scroll: 回溯可见系数（老用户翻评论的概率）
```

## 算法详解

### 第一步：计算渗透率，判断生命周期

**关键洞察**：单看发布时间不够准确，需要结合「阅读量/粉丝数」比值来判断推文处于哪个阶段。

```
渗透率 = 当前阅读量 / 推主粉丝数

场景对比：
- 100万粉丝，1万阅读 → 渗透率 1%，刚起步
- 100万粉丝，50万阅读 → 渗透率 50%，已过半
- 1万粉丝，1万阅读 → 渗透率 100%，已见顶
```

#### 生命周期阶段

| 阶段 | 渗透率 | 速度 | 剩余潜力 | 含义 |
|------|--------|------|---------|------|
| 🚀 **launching** | < 5% | - | 90% | 刚起步，潜力巨大 |
| 📈 **accelerating** | 5-30% | 高 | 50-70% | 加速扩散中 |
| 🔝 **peaking** | 30-60% | 中 | 20-40% | 接近高峰 |
| 📉 **decaying** | 60-100% | 低 | 5-15% | 开始衰退 |
| 💀 **dead** | >100% 或 >24h | - | <2% | 已凉 |

#### 速度计算

```
速度 = 阅读量 / 发布小时数
期望速度 = 粉丝数 × 0.1（假设1小时触达10%粉丝是正常速度）
速度比 = 实际速度 / 期望速度
```

---

### 第二步：计算未来增量 (V_future)

```
理论天花板 = 粉丝数 × 病毒系数
剩余空间 = 天花板 - 当前阅读量
V_future = 剩余空间 × 剩余潜力 × 实现率
```

#### 病毒系数 [v1.1 更新：加权互动率]

**v1.0 旧版**：
```
互动率 = (点赞 + 转发 + 评论) / 阅读量
```

**v1.1 新版**（基于 X 算法权重）：
```
加权互动数 = (Like × 1) + (Reply × 20) + (Retweet × 30)
加权互动率 = 加权互动数 / 阅读量
病毒系数 = 1 + min(4, 加权互动率 × 5)
```

**权重说明**（来自 X 推荐算法）：
| 互动类型 | 权重 | 原因 |
|---------|------|------|
| Retweet/Quote | 30 | 核心传播引擎，带来新受众 |
| Reply | 20 | 提升排名，触发通知 |
| Like | 1 | 辅助信号，权重最低 |

#### 半衰期（时间衰减兜底）

| 推主粉丝量 | 半衰期 | 说明 |
|-----------|--------|------|
| > 100万 | 6 小时 | 大V长尾效应明显 |
| > 10万 | 3 小时 | 中V |
| > 1万 | 1.5 小时 | 小V |
| < 1万 | 45 分钟 | 普通用户 |

```
时间衰减系数 = 0.5 ^ (小时数 / 半衰期)
```

---

### 第三步：计算排名转化率 (R_rank)

你的评论能排第几，决定你能分到多少新流量。

#### 排名得分计算 [v1.2 更新：降低粉丝权重]

```
排名得分 = 粉丝分 + 蓝标分 + 早入场分

v1.1 旧版：
  粉丝分 = min(0.25, log10(你的粉丝数) / 15)
  蓝标分 = 0.20
  早入场分: <5 → 0.25, <20 → 0.15, <50 → 0.08, ≥50 → 0.02

v1.2 新版（降低粉丝权重，提高时机权重）：
  粉丝分 = min(0.15, log10(你的粉丝数) / 20)  ← 降低 40%
  蓝标分 = 0.25  ← 提高 25%
  早入场分:
    - 评论数 < 5:  +0.30  ← 提高
    - 评论数 < 20: +0.18
    - 评论数 < 50: +0.10
    - 评论数 ≥ 50: +0.03
```

**v1.2 调整理由**：
- 粉丝数量在 X 算法中不是最关键因素
- 入场时机（早评论）和蓝标认证更重要
- 降低粉丝权重让小号也有机会获得曝光

#### 排名位置预测

```
预测排名 = 当前评论数 × (1 - 排名得分) + 1
```

#### 排名转化率 [v1.1 更新：分段函数]

**v1.0 旧版**：
```
R_rank = 0.20 × 0.72^(排名-1)  // 衰减过快
```

**v1.1 新版**（分段平滑）：
```
Top 1-3 (黄金区):
  - 第1名: 25%
  - 第2名: 18%
  - 第3名: 12%

Top 4-10 (白银区):
  - R_rank = 0.08 × 0.75^(排名-4)

Top 11-30 (青铜区):
  - R_rank = 0.02 × 0.80^(排名-10)

30+ (折叠区):
  - R_rank = 0.5%（几乎只有回溯流量）
```

---

### 第四步：蓝标门槛 (B_gate) [v1.1 新增]

**关键洞察**：在 X 的算法中，非蓝标评论会被折叠到 "Show more replies"，几乎不可见。蓝标不是加分项，而是**入场券**。

```
B_gate = isBlueVerified ? 1.0 : 0.15

含义：
- 蓝标用户：获得完整曝光机会
- 非蓝标用户：曝光直接砍掉 85%
```

**影响计算**：
```
实际曝光 = 理论曝光 × B_gate
```

---

### 第五步：账号健康度 (A_health) [v1.1]

**关键洞察**：新账号会触发 Shadowban（影子禁令），即使有蓝标也会被限流。

```typescript
accountAgeDays = (当前时间 - 账号创建时间) / 天

A_health 计算：
  - 账号 < 7 天:   0.10（新号基本没曝光）
  - 账号 < 30 天:  0.40（1个月内受限）
  - 账号 < 90 天:  0.70（3个月内略受限）
  - 账号 ≥ 90 天:  1.00（老号正常）
```

**示例**：
| 账号 | 创建时间 | 账号年龄 | A_health |
|------|---------|---------|----------|
| Jack | 2020-05-18 | ~5年 | 1.0 |
| 新测试号 | 2025-01-15 | 14天 | 0.4 |
| 刚注册 | 2025-01-25 | 4天 | 0.1 |

---

### 第六步：媒体类型加成 (M_media) [v1.2 新增]

**关键洞察**：带视频或图片的推文更容易获得曝光，评论也会受益于更高的流量。

```typescript
M_media 计算：
  - video（视频/GIF）: 1.50（+50% 加成）
  - image（图片）:     1.20（+20% 加成）
  - none（纯文字）:    1.00（无加成）
```

**媒体类型检测**：
```
优先级：extended_entities.media > entities.media > card

视频类型：
  - type === 'video'
  - type === 'animated_gif'
  - card.name 包含 'player' 或 'video'（YouTube 等嵌入）

图片类型：
  - type === 'photo'
  - 其他有 media 但非视频的情况
```

**影响计算**：
```
实际曝光 = 理论曝光 × M_media

示例（同一场景）：
- 纯文字推文: 1,000 曝光
- 带图片推文: 1,200 曝光 (+20%)
- 带视频推文: 1,500 曝光 (+50%)
```

---

### 第七步：计算回溯流量 (V_backlog)

已经看过推文的用户，有多少会往下翻看到你的评论。

```
回溯系数 = 0.015 × 0.85^(小时数/4)

V_backlog = 当前阅读量 × 回溯系数

示例：
- 刚发布: 1.5% 的老用户会翻评论
- 4小时后: 1.3%
- 12小时后: 0.9%
- 24小时后: 0.6%
```

---

### 第八步：计算范围（置信区间）

由于存在不确定性，输出范围而非单一数值。

#### 不确定性因子

| 置信度 | 条件 | 下限系数 | 上限系数 |
|--------|------|---------|---------|
| 高 | <1h 且 >500阅读 | 0.7 | 1.4 |
| 中 | <4h 且 >200阅读 | 0.5 | 1.8 |
| 低 | 其他 | 0.3 | 2.5 |

#### 最终计算

```
期望值 = (V_future × R_rank) × B_gate × A_health × M_media + V_backlog
最小值 = (V_future × 下限系数 × 悲观排名) × B_gate × A_health × M_media + (V_backlog × 0.5)
最大值 = (V_future × 上限系数 × 乐观排名) × B_gate × A_health × M_media + (V_backlog × 1.5) × 1.2
```

## 输出结构

```typescript
interface ExposurePrediction {
  // 曝光范围
  minImpressions: number;      // 保守估计
  expectedImpressions: number; // 期望值
  maxImpressions: number;      // 乐观估计

  // 生命周期
  phase: 'launching' | 'accelerating' | 'peaking' | 'decaying' | 'dead';

  // 评估
  timing: 'golden' | 'good' | 'late' | 'dead';
  confidence: 'high' | 'medium' | 'low';
  score: number;               // 0-100 综合评分

  // 账号状态 [v1.1 新增]
  accountHealth: number;       // 0.1-1.0
  blueGateApplied: boolean;    // 是否受蓝标门槛影响

  // 展示
  rangeText: string;           // "200 - 500" 格式化
  recommendation: string;      // 行动建议
}
```

---

## 综合评分计算

```
总分 = 时间分(35) + 曝光分(35) + 热度分(20) + 互动分(10)

时间分:
  - golden: 35
  - good: 25
  - late: 10
  - dead: 2

曝光分 = min(35, log10(预期曝光) / 3 × 35)
热度分 = min(20, log10(当前阅读) / 5 × 20)
互动分 = min(10, 加权互动率 × 100)  [v1.1 更新]
```

### ✅ 已纳入模型

| 因素 | 来源 | 作用 | 版本 |
|------|------|------|------|
| 阅读量/粉丝比（渗透率） | 计算 | 判断生命周期阶段 | v1.0 |
| 阅读量/时间（速度） | 计算 | 判断是否在加速 | v1.0 |
| 推主粉丝数 | API | 理论天花板 + 半衰期 | v1.0 |

| 评论数 | API | 竞争程度 → 排名 | v1.0 |

| 推主回复习惯 | 无法预测 |
