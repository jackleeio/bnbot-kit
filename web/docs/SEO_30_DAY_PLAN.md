# BNBot 30天 SEO 冲刺计划（执行版）

> 目标：提升 bnbot.ai 的收录、排名与稳定自然流量。  
> 当前基础：已具备 /blog、自动生成、基础元数据、sitemap。

## 总体KPI（30天）
- 索引页数提升（GSC）
- 非品牌词点击数提升
- Top 20 关键词数量提升
- 文章平均 CTR 提升

## 第1周：地基强化（技术+结构）
### Day 1
- 建立关键词地图（5大主题：AI/Web3/出海/独立开发/一人公司）
- 每个主题定 10 个主词 + 30 个长尾词

### Day 2
- 检查 canonical、title、meta description 统一策略
- 每篇文章统一 frontmatter 规范

### Day 3
- 完成栏目页：/blog/ai /blog/web3（已完成）
- 增加栏目内链

### Day 4
- 检查 JSON-LD：BlogPosting + FAQ（已完成）
- 验证结构化数据格式

### Day 5
- 优化图片策略：每篇封面图 + alt + 统一命名

### Day 6
- 建立内容质量检查清单（来源、观点、行动建议、FAQ）

### Day 7
- 周复盘：哪些文章收录最快、哪些标题点击更高

## 第2周：内容密度与专题聚合
### Day 8-10
- 建 2 个 Pillar 页面：
  - AI Automation Guide
  - Web3 Growth Playbook

### Day 11-14
- 日更继续（5篇分时段）
- 每篇强制加 Sources（>=2条官方/权威来源）
- 每篇至少 3 条站内内链

## 第3周：分发与外链
### Day 15-17
- 建立跨平台分发模板（Medium / X / LinkedIn / 社区）
- 每篇生成：长文版 + 精简版 + 线程版

### Day 18-21
- 每天至少1篇同步分发到外部平台
- 所有外部内容指回主站 canonical 版本

## 第4周：优化迭代
### Day 22-25
- 对已有文章做二次更新（补新数据、补FAQ、补内链）

### Day 26-28
- 做 CTR 优化（重写低点击标题与描述）

### Day 29
- 合并/下线薄内容，避免关键词互相竞争

### Day 30
- 总复盘 + 下月选题策略更新

---

## 每篇文章发布标准（必须满足）
1. 标题含主关键词
2. Meta Description 120-160字
3. 正文有可执行建议（不是纯搬运）
4. Sources >= 2
5. FAQ >= 2
6. 内链 >= 3
7. 有封面图

## 分发策略（主站优先）
- 主站：完整版本（SEO资产归主站）
- Medium：改写版 + canonical（或导流链接）
- Substack：周报/观点版，承接订阅
- Dev.to / Hashnode：技术实操版（开发者流量）
- LinkedIn：专业版摘要 + 主站链接
- Mirror（Web3主题时）：Web3定向分发
- X：由外部 agent 负责（仅保留文案生成）

## 平台配置优先级（先配好再自动化）
1. Medium
2. Substack
3. Dev.to
4. Hashnode
5. LinkedIn
6. Mirror
